# 1. Introduction
Since we are making this as part of a learning program, I am intentionally making the project simple in terms of scope and depth. While I have previous experience in data science via logit modelling of travel behavior, this project is intended to be a shorter exercise in practicing the tools provided by the Cisco Networking Academy program.

All data sources will be public and is whatever I am able to search via Google, and maybe some secondary data via research papers available publicly. AI work in this project is constrained mainly for quick checking/scraping available datasets, or proofreading the writing for grammar errors. 

I also may want to use the data in the future for a 4 step activity model on opening new railway lines across the metro, or just opening new feeder transpo routes to the station, with regards to their viability/marketability.

# 2. Scope
The project will cover all current railway lines as of June 2026. To keep the data set minimal, I will cover data available from 1980-2026 as publicy available via FOI.gov searchable data, DOTr and agency transparency data, and other sources as may be identified throughout this project. I will not scrape data by building a web scraper, or communicate with offices to ask for data. As such, my analysis will be limited in this sense.
